# Bug: `and`/`or` Short-Circuit — FULLY FIXED

## Status: FIXED (Mar 2, 2026)

Both the original short-circuit bug and the regalloc crash are now resolved.

### Bug 1: No short-circuit evaluation (FIXED)

**Problem:** `lowerBinary` in `lower.zig` evaluated both operands unconditionally before creating the IR node. The right operand's side effects (slice bounds checks) were emitted eagerly into the current block, before the SSA builder's `convertLogicalOp` could defer them behind a branch.

**Fix:** Added `lowerShortCircuit()` in `lower.zig` (Go reference: `ssagen/ssa.go:3398-3442` OANDAND/OOROR expr path). Same deferred-evaluation pattern as `lowerCompoundOrelse` — evaluates left, creates conditional blocks, defers right-side lowering to the appropriate block. Tests: `test/e2e/features.cot` (5 tests).

### Bug 2: Regalloc crash with multiple short-circuit `and` (FIXED)

**Problem:** Functions with 3+ short-circuit `and` expressions with slice indexing caused a compiler crash: `[liveness] EntryLivein error: entry block has live-in vregs`. The register allocator's liveness analysis found values that appeared live at the entry block — impossible in correct SSA.

**Root cause:** `emitTerminator` in `ssa_to_clif.zig` misidentified bounds check fail blocks. These blocks (generated by slice indexing) contain panic call instructions followed by `wasm_unreachable`, have no SSA successors, and are dead ends. The old `is_trap_block` check required ALL values in the block to be `wasm_unreachable`, but bounds check fail blocks also have call instructions for the panic handler. Since the check failed, the code fell through to the "jump to next layout block" path, creating false control flow edges between unrelated blocks. This let the liveness analysis propagate values from conditional short-circuit blocks through the spurious edges to the entry block.

**Fix:** Changed `is_trap_block` detection from "all values are `wasm_unreachable`" to "any value is `wasm_unreachable`". A block with no successors that contains `wasm_unreachable` is always a dead end — it must emit `trap`, not fall through.

**File:** `compiler/codegen/native/ssa_to_clif.zig`, `emitTerminator` function (~line 1296).

**Tests:** 71/71 test files pass. Complex reproducer (3 short-circuit `and` expressions with slice indexing in a parsing function) verified working.
