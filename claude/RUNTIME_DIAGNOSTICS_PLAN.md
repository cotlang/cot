# Runtime Diagnostics Enhancement Plan

**Date:** 2026-03-16 (updated end of session)
**Context:** Self-hosting dogfooding exposed that native binary crashes are nearly impossible to diagnose.
**Goal:** Any crash in a Cot native binary should immediately print a stack trace with file:line information.

## Implementation Status

| Phase | Status | Notes |
|-------|--------|-------|
| Phase 1: Signal handlers | ✅ DONE | `signal_native.zig`: `__cot_signal_handler` + `__cot_install_signals` |
| Phase 2: @trap file:line | ✅ DONE | Shows source location before halt. Cross-file generics show filename+offset |
| Phase 3: Stack traces | ⚠️ PARTIAL | `backtrace_symbols_fd` works for @panic but removed from @trap (can SIGBUS on corrupt heap) |
| Phase 4: Symbol resolution | ✅ DONE | `backtrace_symbols_fd` resolves ASLR-aware function names |

**Known issues:**
- @trap backtrace removed because `backtrace_symbols_fd` crashes when heap is corrupt (ARC use-after-free). Only @panic gets backtraces now.
- Cross-file generic @trap shows `filename:offset(N)` instead of `filename:line` (no source text for imported files in lowerer)
- Signal handler can't print backtrace (not async-signal-safe)

## Architecture: Three Layers

### Layer 1: Signal Handler for Native Binaries
**Reference:** Go `runtime/signal_unix.go`, Zig `std.debug.panicImpl`

Native binaries need the same signal handler that Wasm wrappers already have. When a native program hits SIGSEGV/SIGILL/SIGBUS/SIGFPE/SIGABRT, the handler should:

1. Print signal name and faulting address
2. Walk the frame pointer chain
3. For each frame, print the return address
4. Exit with signal code

**Implementation:**
- Generate a `_cot_install_signal_handler` function in `arc_native.zig` or a new `runtime_native.zig`
- Emit a call to it at the start of `main()` (same as `__cot_init_globals`)
- Handler uses `sigaction()` with `SA_SIGINFO` to get `si_addr` and `uc_mcontext` (for PC)
- Frame walk: start from FP register, follow `*(fp)` chain, print each `*(fp+8)` (return address)

**Output format (Go-style):**
```
fatal error: SIGSEGV
goroutine 1 [running]:
  0x100012abc  TypeRegistry_makePointer+0x1c4
  0x100015def  codegen.wasm.driver.generateAndAddFunc+0x38
  0x10001a012  main+0x240
```

**Effort:** Medium. Signal handler code already exists in `driver.zig` for Wasm wrappers — port it to a native runtime function. Frame walk is ~20 lines of assembly/C.

### Layer 2: PC-to-Source Mapping (DWARF Lookup)
**Reference:** Zig `std.debug.getSelfDebugInfo`, Go `runtime.funcname`

The signal handler from Layer 1 prints raw PCs. To get file:line, the runtime needs to:

1. At link time: embed the `.debug_line` section in the binary (already done for macOS)
2. At runtime: parse the embedded DWARF line table
3. For each PC: binary search the line table to find file:line

**Implementation:**
- DWARF line table is already generated by `dwarf.zig` and written by `macho.zig`
- Add a runtime function `_cot_lookup_line(pc: usize) -> (file: string, line: u32)`
- This function reads the DWARF section from the binary (via mmap or embedded pointer)
- Parses the line number program (state machine per DWARF spec)
- Binary searches for the PC

**Output improvement:**
```
fatal error: SIGSEGV
  self/frontend/ssa_builder.cot:1230  TypeRegistry_makePointer
  self/codegen/wasm/driver.cot:354    generateAndAddFunc
  self/main.cot:180                    cmdBuildWithOutput
```

**Effort:** High. DWARF parsing is complex (~200 lines). Can be simplified by using a custom, simpler line table format instead of full DWARF.

### Layer 3: Debug Build Mode
**Reference:** Zig `-ODebug` vs `-OReleaseFast`, Rust `debug_assertions`

Wire the existing `--release` CLI flag to control:

| Feature | Default (debug) | `--release` |
|---------|-----------------|-------------|
| Bounds checks in stdlib | `@panic` with message | `@trap()` (fast) |
| `@assert` behavior | `@panic` with file:line | `@trap()` |
| Frame pointers | Preserved | Omitted (more registers) |
| DWARF debug info | Emitted | Omitted (smaller binary) |
| Signal handlers | Installed | Not installed |
| Stack trace on panic | Printed | Not printed |

**Implementation:**
- `lower.zig`: check `self.release_mode` when lowering `@trap` / `@assert`
- `driver.zig`: pass release flag through to codegen
- `isa/aarch64/abi.zig`: `preserve_frame_pointers = !release_mode`
- `stdlib/list.cot`, `stdlib/map.cot`: can't easily change at compile time — need `@comptime_if` or conditional compilation. Alternative: the compiler replaces `@trap()` with `@panic(...)` in debug mode during lowering.

**Effort:** Low-Medium. The `--release` flag is already parsed. Wiring it requires touching ~5 files.

## Execution Order

### Phase 1: Signal Handler for Native (Immediate Impact)
**Files to modify:**
- `compiler/codegen/native/io_native.zig` — add `_cot_install_signal_handlers` CLIF function
- `compiler/driver.zig` — call it at program start (same pattern as `__cot_init_globals`)
- `compiler/codegen/native/isa/aarch64/` — ARM64 signal handler assembly (sigaction + frame walk)
- `compiler/codegen/native/isa/x64/` — x64 signal handler (same pattern)

**Reference code to port:**
- `compiler/driver.zig:3312-3335` (Wasm wrapper signal handler) — port to CLIF IR
- Go `runtime/signal_unix.go:sigtrampgo()` — signal trampoline pattern

**Verification:**
```bash
# Before: native crash shows nothing
cot run test_crash.cot  # just "Segmentation fault"

# After: native crash shows signal + PCs
cot run test_crash.cot
# fatal error: SIGSEGV
#   pc=0x100012abc addr=0x0
#   0x100012abc
#   0x100015def
#   0x10001a012
```

### Phase 2: @trap → @panic in Debug Mode (Quick Win)
**Files to modify:**
- `compiler/frontend/lower.zig` — when `!release_mode`, lower `@trap()` as `@panic("trap at file:line")`
- `compiler/cli.zig` / `compiler/main.zig` — thread `--release` flag to lowerer
- `compiler/driver.zig` — pass release flag

**Verification:**
```bash
# Before: List.get bounds check gives SIGILL
cot run test.cot  # just crashes

# After: bounds check gives meaningful error
cot run test.cot
# stdlib/list.cot:55: panic: trap
# (or ideally: "index out of range")
```

### Phase 3: Stack Trace on @panic (Full Diagnostics)
**Files to modify:**
- `compiler/codegen/native/arc_native.zig` or new `stack_trace_native.zig` — frame walk + print
- `compiler/frontend/lower.zig` — `@panic` calls stack trace printer before exit
- Frame walk: read FP chain, print each return address

**Reference:** Go `runtime.traceback()`, Zig `std.debug.dumpStackTrace()`

**Verification:**
```bash
cot run test.cot
# test.cot:5: panic: index out of range
#   0x100012abc
#   0x100015def
#   0x10001a012
```

### Phase 4: DWARF Source Mapping (Full Source-Level Traces)
**Files to modify:**
- New `compiler/codegen/native/debug_info.zig` — DWARF line table parser
- Integrate with stack trace printer from Phase 3

**This is a stretch goal.** Phases 1-3 give 90% of the value. Phase 4 converts hex PCs to file:line, which is a nice-to-have.

## What NOT to Do

- Do NOT implement a full debugger (gdb/lldb integration). That's a post-1.0 feature.
- Do NOT implement memory profiling or race detection yet. Focus on crash diagnostics.
- Do NOT change the stdlib `@trap()` calls to `@panic()` manually. The compiler should do this transformation in debug mode.
- Do NOT block self-hosting on this work. These improvements help diagnose bugs faster, but the selfcot bugs can still be found with the existing @panic-probe approach.

---

## Relationship to Self-Hosting

This work is **parallel to** self-hosting, not sequential. Both tracks advance the compiler:

- **Self-hosting track:** Fix the SSA builder bug (empty List(int).get at idx=0), then build progressively larger files until main.cot compiles.
- **Diagnostics track:** Add signal handlers and stack traces so future bugs are trivially diagnosable.

The diagnostics track should be done first because it makes the self-hosting track dramatically faster. Every bug found by selfcot dogfooding becomes a 5-minute fix instead of a 4-hour investigation.
